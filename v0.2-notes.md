# v0.2 Notes

Goal: Google-only Supabase Auth + business onboarding + backend-enforced multi-tenant foundation.

Decisions:
- Use Google OAuth only; no email/password signup in v0.2.
- Use backend-only tenant enforcement for v0.2, no RLS policies yet.
- Keep the public call demo separate at /demo.
- Add business profile settings only; advanced services/hours/rules editing comes later.

Assumptions:
- Branch is v0.2.
- Public demo rows use business_id = null.
- SaaS rows always use a verified business_id.
- Do not add Stripe, Twilio, invoices, or full CRM in this phase.
